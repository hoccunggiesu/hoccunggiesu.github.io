# Đóng góp cho Dự án Học Cùng Giêsu

Cảm ơn bạn đã quan tâm đóng góp cho Học Cùng Giêsu! 🎉  
Dù bạn là lập trình viên, người viết nội dung, giáo lý viên, tu sĩ hay chỉ là một người yêu mến dự án, chúng tôi đều rất hoan nghênh sự tham gia của bạn.

## Các cách đóng góp

Bạn có thể giúp dự án bằng nhiều cách:

- 📝 **Viết nội dung**: thêm thông tin dòng tu, chủng viện, câu chuyện ơn gọi, bài học giáo lý, tiểu sử các thánh.
- 🐛 **Báo cáo lỗi**: sai chính tả, sai thông tin, lỗi hiển thị trên website.
- 💡 **Đề xuất tính năng**: ý tưởng mới cho website, trò chơi, bài học, cách tra cứu.
- 🛠️ **Sửa code**: cải thiện giao diện, thêm chức năng tìm kiếm, tối ưu hiệu năng.
- 🎨 **Thiết kế**: đóng góp hình ảnh, logo, icon cho dự án.

## Trước khi bắt đầu

- Đọc [README.md](./README.md) để hiểu tổng quan dự án.
- Đọc [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md) – chúng tôi muốn xây dựng một cộng đồng vui vẻ, tôn trọng lẫn nhau.
- Nếu bạn dự định thay đổi lớn (cấu trúc thư mục, giao diện, cách thức hoạt động), vui lòng **tạo issue thảo luận trước** để mọi người cùng góp ý.

## Quy trình đóng góp

### Đối với nội dung (các file trong `content/`)

1. Fork repo và clone về máy.
2. Tạo nhánh mới.
3. Vào `content/`, sao chép file mẫu từ `0. temple/`, điền thông tin theo đúng template.
4. Commit với thông điệp rõ ràng, ví dụ: *"Thêm dòng Đa Minh (OP) – nhánh Nam"*.
5. Đẩy lên GitHub và tạo Pull Request.

Chi tiết hơn có trong [README.md](./README.md#hướng-dẫn-đóng-góp).

### Đối với code (HTML, CSS, JS)

1. Fork và clone repo.
2. Cài đặt môi trường phát triển theo [HOW_TO_BUILD.md](./HOW_TO_BUILD.md).
3. Tạo nhánh mới.
4. Thực hiện thay đổi, đảm bảo không phá vỡ chức năng hiện tại.
5. Nếu thêm tính năng mới, vui lòng thêm mô tả ngắn gọn trong commit và trong Pull Request.
6. Gửi Pull Request.

### Gửi Issue

- Đối với lỗi: mô tả rõ **lỗi gì**, **xảy ra ở đâu** (trang nào, file nào), **các bước để tái hiện**.
- Đối với đề xuất: mô tả **tính năng mới hoạt động thế nào**, **tại sao nên có**, và nếu được, **gợi ý cách làm**.

## Quy trình phát hành (Release)

Dự án hiện triển khai tự động qua **GitHub Pages**. Mỗi khi code được merge vào nhánh `main`, website sẽ tự cập nhật sau vài phút.  
Các bản phát hành có đánh phiên bản (tag) sẽ được ghi chú trong [CHANGELOG.md](./CHANGELOG.md).

## Liên hệ

Nếu bạn có thắc mắc hoặc cần hỗ trợ, hãy:
- Tạo một issue mới.
- Liên hệ qua email của người bảo trì (sẽ cập nhật sau).
